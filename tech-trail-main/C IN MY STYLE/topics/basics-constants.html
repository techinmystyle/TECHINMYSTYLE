Constants in C Programming

Overview

A constant is a fixed value that does not change during program execution. Constants are used when a value must remain the same throughout a program, improving readability, safety, and maintainability.

Why use constants?

To protect important values (like Pi or maximum limits) from accidental modification.

Types of Constants

  • Integer Constants: Whole numbers (e.g., 10, -25)
  • Floating-point Constants: Real numbers (e.g., 3.14, -2.718)
  • Character Constants: A single character enclosed in single quotes (e.g., 'A')
  • String Constants: A sequence of characters enclosed in double quotes (e.g., "Hello")
  • Enumeration Constants: Named constants created using enum

Defining Constants

constants.c
#include <stdio.h>

#define PI 3.14159      // Macro constant
const int MAX = 100;    // Constant variable

int main() {
    printf("PI = %f\\n", PI);
    printf("MAX = %d\\n", MAX);

    // Uncommenting below line will cause an error
    // MAX = 200;

    return 0;
}
  • #define – Preprocessor directive for macro constants
  • const – Declares a constant variable (type-safe)

Difference: #define vs const

Aspect #define const
Type Safety No (text substitution) Yes
Scope Global Block/function-level
Debuggable No (disappears in compilation) Yes
Preferred In Old-style C Modern C with type checking

Real-World Applications

Geometry Tools

Using constants like PI to ensure precise calculations in formulas.

Security Limits

Storing max login attempts, timeout durations using constants.

Game Config

Player speed, game limits, and difficulty levels can be constants.

Interview Questions

What happens if you try to change a constant?

The compiler will throw an error if you try to modify a const variable.

When should we use const over #define?

const should be preferred in modern C as it is type-safe and has better scope control.
