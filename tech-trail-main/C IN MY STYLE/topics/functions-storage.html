Function Storage Classes in C

Overview

In C, a storage class defines the scope (visibility), lifetime, and linkage of variables and functions. For functions, the primary concern is how the storage class affects linkageâ€”whether the function is visible outside the file it is declared in.

Quick Fact

Unlike variables, functions can only have two storage classes: extern and static.

Function Storage Classes

extern

This is the default. A function declared without a storage class is assumed to have external linkage, meaning it can be used across multiple files.

static

Limits the function's visibility to the file it is defined in. Useful for encapsulation and avoiding naming conflicts.

Examples

1. External Function (default)

// file1.c
#include <stdio.h>

void display() {
    printf("Hello from file1.c\\n");
}
// file2.c
void display();  // extern by default

int main() {
    display();  // Calls function from file1.c
    return 0;
}

2. Static Function

// file1.c
#include <stdio.h>

static void greet() {
    printf("Hello from static function\\n");
}

int main() {
    greet();
    return 0;
}

This function greet() cannot be called from another file.

Real-World Applications

Library Development

Use static for helper functions that should not be exposed to users of the library.

Modular Programming

Use extern to allow functions to be shared across modules.

Interview Questions

What is the default storage class of a function?

Functions are extern by default, meaning they can be called from other files.

What is the purpose of declaring a function as static?

Declaring a function static limits its scope to the file it is defined in, preventing conflicts and ensuring encapsulation.
