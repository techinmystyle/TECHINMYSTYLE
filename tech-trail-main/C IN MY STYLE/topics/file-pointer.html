File Pointers in C

Overview

In C, a file pointer is a pointer of type FILE* used to manage and manipulate files. It points to a structure that contains information about the file being accessed, such as its current position, access mode, and more.

Definition

FILE *fp; – Declares a file pointer that can be used with C standard I/O functions like fopen(), fclose(), fscanf(), etc.

Declaring and Using a File Pointer

#include <stdio.h>

int main() {
    FILE *fp;
    fp = fopen("data.txt", "r"); // Open file in read mode

    if (fp == NULL) {
        printf("File could not be opened.\n");
        return 1;
    }

    char ch;
    while ((ch = fgetc(fp)) != EOF) {
        putchar(ch);
    }

    fclose(fp); // Always close the file
    return 0;
}

Explanation:

  • FILE *fp - Declares the file pointer.
  • fopen() - Opens the file and returns a pointer to it.
  • fgetc(fp) - Reads character-by-character using the file pointer.
  • fclose(fp) - Closes the file.

Common Operations with File Pointers

  • fopen() – Opens a file and returns a file pointer.
  • fclose() – Closes the file using the file pointer.
  • fgetc(), fputc() – Read/write a single character.
  • fgets(), fputs() – Read/write a string.
  • fprintf(), fscanf() – Formatted input/output to/from files.
  • feof() – Check for end of file.

Real-World Applications

Text Editors

Use file pointers to read/write contents to disk efficiently.

Data Loggers

Append and read logs using file pointer operations.

Configuration Parsers

Parse config files using fgets/fscanf with a file pointer.

Interview Questions

What is a file pointer in C?

A file pointer is a pointer to a FILE structure, used to manage input/output operations on files.

Why is fclose() important?

fclose() ensures that all data is flushed from buffers and the file is properly closed to avoid memory leaks or corruption.

How do you check if file was opened successfully?

By checking if the file pointer is NULL after calling fopen().
