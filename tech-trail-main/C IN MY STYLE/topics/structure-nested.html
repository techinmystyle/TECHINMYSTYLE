Understanding Nested Structures in C

Overview

A nested structure in C refers to including one structure within another. This allows developers to logically group sub-records or detailed information inside larger data entities. It is particularly helpful in modeling real-world relationships, such as a student having a nested date of birth structure.

Tip

Use nested structures to simplify hierarchical data and enhance data grouping and management.

Declaration and Initialization

Here is how you declare and initialize a nested structure:

nested_struct.c
#include <stdio.h>

struct Date {
    int day, month, year;
};

struct Student {
    char name[30];
    int roll;
    struct Date dob; // Nested structure
};

int main() {
    struct Student s1 = {"Alice", 101, {12, 8, 2002}};

    printf("Name: %s\\n", s1.name);
    printf("Roll: %d\\n", s1.roll);
    printf("DOB: %02d-%02d-%04d\\n", s1.dob.day, s1.dob.month, s1.dob.year);

    return 0;
}

Explanation:

  • struct Date is defined first and then used inside struct Student.
  • s1 is initialized with nested values using curly braces.
  • Access nested members using dot notation like s1.dob.day.

Accessing Nested Members

To access members of a nested structure, use the dot operator (.) twice, chaining through the nested member:

access_nested.c
#include <stdio.h>

struct Address {
    char city[20];
    int pincode;
};

struct Employee {
    char name[20];
    struct Address addr; // Nested structure
};

int main() {
    struct Employee e1 = {"Rahul", {"Mumbai", 400001}};

    printf("Employee: %s\\n", e1.name);
    printf("City: %s\\n", e1.addr.city);
    printf("Pincode: %d\\n", e1.addr.pincode);

    return 0;
}

Interview Questions

What is a nested structure?

A nested structure is a structure that contains another structure as one of its members.

Can nested structures be passed to functions?

Yes, like other structures, nested structures can be passed to functions by value or by reference.

How to access a member of a nested structure?

Use dot notation: outer.inner.member. Example: e1.addr.city.
