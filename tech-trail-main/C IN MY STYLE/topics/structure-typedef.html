Using typedef with Structures in C

Overview

In C, the typedef keyword is used to assign alternative names (aliases) to existing data types. When combined with struct, it simplifies syntax and improves code readability by allowing you to omit the struct keyword when declaring structure variables.

Tip

Use typedef with structures to make the code cleaner and easier to maintain, especially when dealing with complex or frequently used structures.

Declaration and Usage

The following example shows how to declare a structure and create a typedef alias for it:

typedef_struct.c
#include <stdio.h>

// Traditional structure declaration with typedef
typedef struct {
    int id;
    char name[30];
    float salary;
} Employee;

int main() {
    Employee e1 = {101, "Neha", 55000.0};

    printf("ID: %d\\n", e1.id);
    printf("Name: %s\\n", e1.name);
    printf("Salary: %.2f\\n", e1.salary);

    return 0;
}

Explanation:

  • typedef struct { ... } Employee; creates an alias Employee for the unnamed structure.
  • No need to use the struct keyword while declaring variables like e1.

Interview Questions

What is typedef in C?

typedef is used to give an alternate name to a data type, including structures, to simplify code.

Why use typedef with structures?

It allows the use of structure variables without repeating the struct keyword, making code cleaner and easier to write.

Can typedef be used with nested structures?

Yes, typedef can be used with both top-level and nested structures for aliasing complex data types.
