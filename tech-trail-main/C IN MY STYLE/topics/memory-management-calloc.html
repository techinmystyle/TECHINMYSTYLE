calloc() in C – Contiguous Memory Allocation

Overview

The calloc() function in C is used to dynamically allocate memory for an array of elements and initializes each block to zero. It's part of the C Standard Library and is often used when you want both allocation and initialization.

Key Point

calloc() differs from malloc() in that it automatically initializes allocated memory to zero.

Syntax

Syntax
void* calloc(size_t num_elements, size_t element_size);

- num_elements: Number of elements to allocate.
- element_size: Size of each element in bytes.
- Returns: A pointer to the allocated memory block or NULL if allocation fails.

Example

This example demonstrates how calloc() allocates and zero-initializes memory:

calloc_example.c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *arr;
    int n = 5;

    arr = (int *)calloc(n, sizeof(int));

    if (arr == NULL) {
        printf("Memory allocation failed!\\n");
        return 1;
    }

    printf("Array values after calloc:\\n");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }

    free(arr);
    return 0;
}

calloc() vs malloc()

Initialization

calloc() initializes memory to 0, malloc() leaves it uninitialized.

Parameters

calloc() takes two arguments: number of elements and size per element.

Performance

calloc() may be slightly slower due to initialization.

Best Practices

  • Always check if the pointer returned by calloc() is NULL.
  • Use free() after you’re done using the dynamically allocated memory.
  • For zero-initialized arrays, calloc() is preferred over malloc().

Interview Questions

What is the difference between malloc() and calloc()?

malloc() allocates uninitialized memory, whereas calloc() allocates and zero-initializes memory.

Why might calloc() be slower than malloc()?

Because calloc() initializes all allocated memory to zero, which adds overhead compared to malloc().
