Command Line Arguments in C

Overview

Command Line Arguments allow users to pass values to a program at the time of its execution from the terminal. They are particularly useful for creating flexible and reusable programs.

Quick Fact

Every C program that uses command-line arguments must define main() with two parameters: int argc and char *argv[].

Syntax

The main function should be defined as:

int main(int argc, char *argv[]) {
    // code
}
  • argc – Argument Count: stores the number of command-line arguments.
  • argv – Argument Vector: stores the actual arguments as strings.

Example Program

cmd_args.c
#include <stdio.h>

int main(int argc, char *argv[]) {
    printf("Number of arguments: %d\\n", argc);
    
    for (int i = 0; i < argc; i++) {
        printf("Argument %d: %s\\n", i, argv[i]);
    }

    return 0;
}

Usage (in terminal):

gcc cmd_args.c -o cmdargs
./cmdargs hello world 123

Output:

Number of arguments: 4
Argument 0: ./cmdargs
Argument 1: hello
Argument 2: world
Argument 3: 123

Real-World Applications

CLI Utilities

Tools like compilers, linters, and file processors use arguments to set options.

Database Scripts

Pass connection strings or queries as arguments to C programs.

Automation

Run batch jobs by passing filenames or parameters via command line.

Interview Questions

What is the use of argc and argv?

argc stores the count of arguments, and argv is an array of strings holding the arguments themselves.

What does argv[0] usually contain?

It usually contains the name of the executable program being run.
