Introduction to Preprocessing in C

Overview

Preprocessing is the first step of the compilation process in C. It is handled by the *C preprocessor*, which processes directives that begin with the # symbol before the actual compilation starts. These directives instruct the compiler to perform specific operations like file inclusion, macro expansion, and conditional compilation.

Quick Fact

Preprocessor directives are not C statements—they do not end with semicolons and are executed before the compiler sees the code.

Common Preprocessor Directives

  • #include — Includes contents of a file.
  • #define — Defines a macro.
  • #undef — Undefines a macro.
  • #ifdef, #ifndef — Conditional compilation based on macro definitions.
  • #if, #else, #elif, #endif — Conditional inclusion of code blocks.

Example: Macro Definition and Conditional Compilation

preprocess_demo.c
#include <stdio.h>
#define PI 3.14
#define DEBUG 1

int main() {
    printf("Value of PI: %f\\n", PI);

    #ifdef DEBUG
        printf("Debugging is enabled\\n");
    #endif

    return 0;
}

Explanation:

  • #define PI 3.14 creates a macro named PI.
  • #ifdef DEBUG checks if DEBUG is defined and includes the code block.

Interview Questions

What is preprocessing in C?

Preprocessing is the first phase in compilation where directives like #include and #define are handled before compiling the source code.

Is #define a C statement?

No, it is a preprocessor directive and not a part of the compiled C code. It is processed before actual compilation begins.

What is the difference between #include <file.h> and #include "file.h"?

<> searches for the header in system directories, while "" searches in the current directory first.
