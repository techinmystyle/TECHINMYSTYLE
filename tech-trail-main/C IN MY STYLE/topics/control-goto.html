Goto Statement in C

Overview

The goto statement in C is used to transfer control to another part of the program unconditionally. Although it can be useful in certain cases, it's generally discouraged due to its potential to make code harder to read and maintain.

Warning

Excessive use of goto can lead to “spaghetti code” — code that’s tangled and difficult to follow.

Syntax

goto label;
// some code
label:
// code to execute when jumped to

Example

#include <stdio.h>

int main() {
    int num = 1;
    if (num == 1) {
        goto skip;
    }

    printf("This line will be skipped.\\n");

    skip:
    printf("Jumped to the label 'skip'.\\n");
    return 0;
}

When to Use goto

  • Breaking out of nested loops
  • Error handling in low-level systems
  • Early exits in complex logic (rare use case)

Real-World Applications

Embedded Systems

Used in microcontroller firmware for error recovery paths.

System Kernels

Early C-based kernels used goto for exception handling.

Security Handling

Critical sections can use goto to exit on error checks.

Interview Questions

Why is goto considered harmful?

It can make control flow difficult to understand and maintain, leading to "spaghetti code."

Is goto ever useful?

Yes, in specific situations like breaking out of deeply nested loops or performing error recovery in low-level code.
