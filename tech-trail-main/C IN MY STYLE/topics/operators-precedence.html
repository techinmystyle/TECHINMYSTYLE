Operator Precedence and Associativity in C

Introduction

In C programming, when multiple operators appear in an expression, their execution is determined by their precedence and associativity. Operator precedence defines the priority of operators, and associativity defines the direction (left to right or right to left) in which an expression is evaluated when operators of the same precedence appear.

Quick Note

int x = 5 + 2 * 3; is evaluated as 5 + (2 * 3) due to the higher precedence of the * operator.

Example Program

This example demonstrates how operator precedence and associativity affect evaluation:

precedence_example.c
#include <stdio.h>

int main() {
    int a = 10, b = 5, c = 2;
    int result;

    result = a + b * c;
    printf("a + b * c = %d\\n", result);  // Expected: 10 + (5 * 2) = 20

    result = (a + b) * c;
    printf("(a + b) * c = %d\\n", result); // Expected: (10 + 5) * 2 = 30

    result = a > b && b > c;
    printf("a > b && b > c = %d\\n", result); // Expected: 1

    return 0;
}

Interview Questions

What is operator precedence?

Operator precedence defines the order in which different operators in an expression are evaluated.

What is the associativity of assignment operators?

Assignment operators (=, +=, -=, etc.) have right-to-left associativity.
