Text File vs Binary File in C

Overview

In C, files can be handled in two main formats: Text files and Binary files. The difference lies in how data is stored and read. Understanding this distinction helps in selecting the appropriate mode for file operations.

Key Difference

Text files store data in readable character format, whereas binary files store data in raw byte format.

Comparison Table

Aspect Text File Binary File
Format Human-readable characters (ASCII) Machine-readable (raw bytes)
Mode "r", "w", "a", "r+", "w+", "a+" "rb", "wb", "ab", "rb+", "wb+", "ab+"
File Size Generally larger due to character encoding Compact since stored in native binary form
Editing Can be opened in a text editor Not readable in text editors
Speed Slower due to parsing of characters Faster read/write operations
Use Cases Logs, reports, config files Images, audio, video, structures

Code Example: Writing Text vs Binary

Text File Example

#include <stdio.h>

int main() {
    FILE *fp = fopen("textfile.txt", "w");
    int a = 10;
    fprintf(fp, "%d", a);
    fclose(fp);
    return 0;
}

Binary File Example

#include <stdio.h>

int main() {
    FILE *fp = fopen("binaryfile.bin", "wb");
    int a = 10;
    fwrite(&a, sizeof(int), 1, fp);
    fclose(fp);
    return 0;
}

Real-World Applications

Text Files

Used for storing logs, reports, and configuration data.

Binary Files

Ideal for multimedia, custom file formats, or storing C structures.

Interview Questions

When would you prefer binary over text files?

When file size and speed are important, or when storing complex data like structures or media.

Can a binary file be opened with Notepad?

No, binary files contain raw bytes which are not human-readable.
