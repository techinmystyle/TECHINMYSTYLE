Recursion in C

Overview

Recursion in C refers to the technique where a function calls itself directly or indirectly. It is a powerful concept used to solve problems that can be broken down into smaller, similar subproblems such as factorial calculation, Fibonacci series, and tree traversals.

Quick Fact

Every recursive function must have a base case to prevent infinite recursion and stack overflow errors.

Syntax of Recursion

void function() {
    // base condition
    if (condition)
        return;

    // recursive call
    function();
}

Example: Factorial Using Recursion

#include <stdio.h>

int factorial(int n) {
    if (n == 0)
        return 1;
    else
        return n * factorial(n - 1);
}

int main() {
    int num = 5;
    printf("Factorial of %d is %d\\n", num, factorial(num));
    return 0;
}

Explanation:

  • Function factorial() calls itself with a smaller value each time.
  • When n == 0, it returns 1 (base case).
  • This halts further recursion and starts resolving the previous calls.

Types of Recursion

  • Direct Recursion: Function calls itself directly.
  • Indirect Recursion: Function calls another function which eventually calls the first function.
  • Tail Recursion: Recursive call is the last operation in the function.

Indirect Recursion Example

#include <stdio.h>

void A(int);
void B(int);

void A(int n) {
    if (n > 0) {
        printf("%d ", n);
        B(n - 1);
    }
}

void B(int n) {
    if (n > 1) {
        printf("%d ", n);
        A(n / 2);
    }
}

int main() {
    A(20);
    return 0;
}

Real-World Applications

Tree Traversals

Preorder, Inorder, and Postorder in binary trees are classic recursive applications.

Backtracking Algorithms

Used in maze-solving, N-Queens, and Sudoku solvers.

Divide & Conquer

Algorithms like Merge Sort and Quick Sort heavily use recursion.

Interview Questions

What is recursion in C?

Recursion is a process where a function calls itself directly or indirectly to solve a problem.

What are the drawbacks of recursion?

Recursive solutions can consume more memory and may lead to stack overflow if the base case is not well-defined.
