#define Preprocessor Directive in C

Overview

The #define directive in C is used to define constant values and macros. It tells the preprocessor to replace all instances of a particular identifier with a specified value or block of code before the program is compiled.

Tip

Unlike variables, constants defined using #define do not use memory and are replaced at compile-time.

Types of Macros

  • Object-like macros: Define a value.
  • Function-like macros: Define a reusable code block with parameters.

Examples

define_examples.c
#include <stdio.h>

#define PI 3.14159
#define SQUARE(x) ((x) * (x))

int main() {
    printf("Value of PI: %f\\n", PI);
    printf("Square of 5: %d\\n", SQUARE(5));
    return 0;
}

Explanation:

  • #define PI 3.14159 defines a constant value for PI.
  • #define SQUARE(x) ((x) * (x)) is a macro function that returns the square of a number.

Real-World Applications

Mathematical Constants

Define commonly used constants for reuse in programs.

PI MAX_VALUE

Reusable Code Blocks

Use function-like macros for small code operations to improve reusability.

SQUARE(x) IS_EVEN(n)

Interview Questions

What is the purpose of #define in C?

The #define directive defines a macro that replaces code tokens with values or blocks of code during preprocessing.

What is the difference between #define and const?

#define is handled by the preprocessor and has no type checking. const is a C keyword with type checking, and it occupies memory.

Can macros be dangerous?

Yes. Improper use, especially with macro functions (like SQUARE(x)), can lead to unintended behaviors due to lack of type checking and side effects.
