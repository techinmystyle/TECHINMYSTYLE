Introduction to Error Handling in C

Overview

Error handling in C is the process of responding to and managing runtime errors that occur while a program is running. Since C does not provide built-in exception handling (like try-catch in other languages), developers must use techniques like return codes, errno, and system-provided error messages to detect and manage errors.

Why It Matters

Ignoring errors can lead to crashes, memory leaks, security vulnerabilities, and undefined behavior. Robust programs anticipate and handle failures gracefully.

Common Types of Errors

  • Compile-time Errors: Syntax or semantic errors found by the compiler.
  • Link-time Errors: Missing function definitions or unresolved symbols during linking.
  • Runtime Errors: Errors during program execution (e.g., divide by zero, file not found).
  • Logical Errors: Code runs without crashing but produces incorrect results.

Basic Error Handling Techniques

  • Return Codes: Functions return values indicating success or failure (e.g., 0 for success, -1 for failure).
  • errno Variable: A global variable that stores error codes after a library function fails.
  • perror() and strerror(): Used to print human-readable error messages.
  • Custom Logging: Printing custom messages for debugging and monitoring.

Simple Example

#include <stdio.h>
#include <errno.h>
#include <string.h>

int main() {
    FILE *fp = fopen("nonexistent.txt", "r");
    if (fp == NULL) {
        printf("Error opening file: %s\\n", strerror(errno));
    } else {
        fclose(fp);
    }
    return 0;
}

Real-World Applications

Database Access

Checks for connection errors and query failures before executing commands.

File Handling

Detects file opening, reading, or writing failures in applications like compilers and editors.

Networking

Handles socket errors, timeouts, and data transfer interruptions.

Interview Questions

Does C have built-in exception handling?

No. C does not support built-in try-catch mechanisms. Developers must use return codes and errno.

What is errno?

errno is a global variable in C that stores the error code set by system calls or library functions when an error occurs.

How is perror() used?

perror() prints a custom error message followed by a description of the last error encountered based on errno.
