Bit Fields in C Programming

Overview

Bit fields in C allow the packing of data in a structure. This is especially useful when memory or data size is a constraint. Using bit fields, you can specify the exact number of bits to be used for a variable within a struct, helping optimize space.

Key Point

Bit fields are ideal for memory-constrained environments such as embedded systems or for implementing flags using fewer bits.

Syntax of Bit Fields

Bit fields are defined inside a struct using a colon (:) followed by the number of bits:

bitfields.c 
#include <stdio.h>

struct Flags {
    unsigned int isVisible : 1;
    unsigned int isEditable : 1;
    unsigned int isDeleted : 1;
};

int main() {
    struct Flags f = {1, 0, 0};
    printf("Visible: %u, Editable: %u, Deleted: %u\n", f.isVisible, f.isEditable, f.isDeleted);
    return 0;
}

Interview Questions

What are bit fields in C?

Bit fields are variables in a struct that occupy a specific number of bits, allowing memory-efficient storage of data.

What is the advantage of using bit fields?

They reduce memory usage by allowing you to store multiple logical/flag values in a compact form.

Can we take address of a bit field variable?

No, you cannot take the address of a bit field because they may not occupy full memory bytes.
