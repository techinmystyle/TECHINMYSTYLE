Difference Between Strings and Characters in C

Overview

In C programming, characters and strings are closely related but fundamentally different. A char holds a single character, while a string is an array of characters ending with a null terminator '\0'.

Key Insight

A string is a collection of characters, but a character is just a single unit. They differ in declaration, usage, and memory handling.

Comparison Table

Aspect char string (char[])
Definition Represents a single character Represents a sequence of characters
Memory Occupies 1 byte Occupies length + 1 bytes (null character)
Syntax char c = 'A'; char str[] = "Apple";
Printing printf("%c", c); printf("%s", str);
Terminator No terminator Ends with '\0'

Example Program

char_vs_string.c
#include <stdio.h>

int main() {
    char c = 'X';             // single character
    char str[] = "Hello";     // string

    printf("Character: %c\\n", c);
    printf("String: %s\\n", str);

    return 0;
}

Real-World Applications

User Input

Character inputs used for menu selection; strings used for names, addresses, etc.

System Commands

Character flags vs command strings in shell interfaces or arguments.

Interview Questions

What is the difference between char and char[] in C?

char holds a single character. char[] (string) holds multiple characters ending with a '\0' terminator.

Can a single character be treated as a string?

No. Strings require a null terminator. A single 'A' is not equivalent to "A".
