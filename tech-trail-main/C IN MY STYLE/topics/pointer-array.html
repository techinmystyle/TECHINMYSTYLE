Pointers and Arrays in C

Overview

Arrays and pointers in C are closely related. The name of an array is essentially a constant pointer to its first element. Pointers can be used to access and manipulate arrays efficiently through pointer arithmetic.

Quick Note

In C, arr[i] is equivalent to *(arr + i).

Key Concepts

  • Array Name: Acts as a constant pointer to the first element.
  • Pointer Arithmetic: Can be used to navigate through array elements.
  • Dynamic Array Access: Achieved by using pointers instead of array indices.
  • Function Arguments: Arrays are passed to functions as pointers.

Example: Array Access via Pointer

array_pointer.c
#include <stdio.h>

int main() {
    int arr[] = {5, 10, 15, 20};
    int *ptr = arr;

    for (int i = 0; i < 4; i++) {
        printf("arr[%d] = %d, *(ptr + %d) = %d\\n", i, arr[i], i, *(ptr + i));
    }

    return 0;
}

Explanation:

  • ptr = arr points to the first element of the array.
  • *(ptr + i) accesses array elements just like arr[i].

Advanced Example: Pointer as Function Argument

pointer_function.c
#include <stdio.h>

void printArray(int *ptr, int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", *(ptr + i));
    }
    printf("\\n");
}

int main() {
    int nums[] = {2, 4, 6, 8};
    printArray(nums, 4);
    return 0;
}

Interview Questions

Is an array name a pointer?

An array name is a constant pointer to its first element. However, it's not a modifiable l-value like a normal pointer.

Can we use pointer arithmetic on arrays?

Yes. You can use *(arr + i) to access elements using pointer arithmetic.

What is the difference between arr and &arr?

arr gives the address of the first element, while &arr gives the address of the entire array block.
