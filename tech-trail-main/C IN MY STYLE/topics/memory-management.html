Introduction to Memory Management in C

Overview

Memory management in C refers to the process of allocating, using, and freeing memory during the execution of a program. C provides both **static** and **dynamic memory management**, giving the programmer fine-grained control over how memory is utilized.

Key Point

Unlike higher-level languages, C does not have garbage collection — it is the programmer’s responsibility to manually manage memory.

Types of Memory in C

  • Stack: Used for static memory allocation (local variables inside functions).
  • Heap: Used for dynamic memory allocation (e.g., using malloc).
  • Data Segment: Stores global and static variables.
  • Code Segment: Contains compiled program instructions (machine code).

Memory Allocation Functions

malloc()

Allocates memory of specified size and returns a void pointer to the first byte.

calloc()

Allocates memory for an array of elements and initializes all bytes to zero.

realloc()

Resizes previously allocated memory block.

free()

Deallocates memory previously allocated using malloc, calloc, or realloc.

Code Example

This example demonstrates memory allocation and deallocation using malloc() and free():

memory.c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *arr;
    int n = 5;

    arr = (int*) malloc(n * sizeof(int)); // allocate memory for 5 integers

    if (arr == NULL) {
        printf("Memory not allocated!\\n");
        return 1;
    }

    for (int i = 0; i < n; i++) {
        arr[i] = i * 2;
    }

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }

    free(arr); // deallocate memory
    return 0;
}

Common Issues in Memory Management

  • Memory leaks: Forgetting to free dynamically allocated memory.
  • Dangling pointers: Using memory that has already been freed.
  • Buffer overflow: Writing outside the allocated memory range.
  • Uninitialized pointers: Using pointers without allocating memory.

Interview Questions

What is the difference between malloc and calloc?

malloc allocates memory without initializing it, whereas calloc allocates and initializes memory to zero.

What happens if you forget to free memory?

This leads to memory leaks, which cause programs to consume more and more memory over time.
